# ダッシュボード最小版 (第 4 グループ 第 1 PR)

## Context

ログイン後のリダイレクト先 `/` が 404 のまま (L-1 検証以来の既知の欠落)。
仕様 §5.6 のダッシュボードはウィジェット 11 種のうち大半が SKYSEA
(installations)・AD (退職者データの実流入)・名寄せ・棚卸し・整合性永続化に
依存する。本 PR は**現存するデータ源だけで意味を持つ 4 ウィジェット**の
最小版を実装し、`/` の 404 を解消する。

## 主要決定

| 項目 | 決定 | 判断 |
|------|------|------|
| 対象ウィジェット | (1) ライセンス保有・過不足 (全社集計、v_license_usage) (2) 承認状況サマリ (default_approval_status 別の製品数) (3) 満了間近ライセンス (90 日以内、期限昇順) (4) 退職者の未解除割当 (仕様 §5.14 のクエリ) | データ源が実在する 4 種のみ。§5.6 の表で全ロール開示 (1)(2)、viewer 以上 (3)、license_manager 以上 (4) |
| 未実装ウィジェット | 画面に出さない (プレースホルダも置かない) | 空枠は運用者を混乱させる。SKYSEA / AD / 棚卸しの各フェーズで追加する (Plan に対応表を記録) |
| ロール別表示 | ウィジェット単位の出し分けのみ (§5.6 の ◯/× 列)。「自部署」の部署スコープは全社表示のまま | 部署スコープ認可は継続負債。現利用者は管理者系のみ |
| ルート | GET `/` = 認証済み全ロール。既存の 404 ハンドラより先に登録 | ログインフローの自然な着地点 |
| 過不足の表示 | product ごとに 保有 / 割当 (count_unit 側) / 不足・超過バッジ。installed は SKYSEA 取込後に意味を持つため列ごと出さない | 現時点で常に 0 の列は誤解の元 |
| 退職者未解除割当 | §5.14 の SQL をそのまま sqlc 化 (user_assignments × users.deactivated_at NOT NULL) | 仕様に SQL が明記されている |
| ナビゲーション | 共通レイアウトのナビに「ダッシュボード」リンクを追加 (先頭) | 到達手段の提供 |
| 新規パッケージ | なし。web ハンドラ + view + sqlc クエリのみ | 集計は SQL で完結する |

## 対象スコープ

- repository: `CountProductsByDefaultApprovalStatus` /
  `ListExpiringLicenses` (90 日・期限昇順・現役のみ) /
  `ListDeactivatedUserAssignments` (§5.14) / 既存 v_license_usage 一覧
  (`ListLicenseUsage :many` 追加)
- web: `dashboard.go` (GET /) + `internal/view/dashboard/`
- layout ナビ更新、handler テスト (ロール別ウィジェット表示 / 非表示)

### 範囲外 (別 PR)

- 名寄せ・未承認/禁止インストール・棚卸し・整合性・推移グラフの各
  ウィジェット (依存フェーズで追加)
- 部署スコープ (自部署) 絞り込み (継続負債)
- general_user 向け表示調整 (AD 後)

## TDD コミット順序

1. `docs(plans): 本 Plan`
2. `feat(repository): ダッシュボード集計クエリ (sqlc)`
3. `test(web): ダッシュボードのウィジェット表示とロール別出し分け (RED)`
4. `feat(web): GET / ダッシュボード最小版 (GREEN)`

## 受け入れ基準

- 全ゲート緑
- 実サーバ: ログイン → `/` が 200 (404 解消)。保有 / 割当 / 過不足、
  承認状況の製品数、満了間近一覧、退職者未解除割当が表示される。
  viewer には退職者ウィジェットが出ない。満了間近は viewer に出るが
  general_user ロールでは出ない (§5.6)
- 過不足: total_count NULL (無制限) の製品が「不足」扱いにならない
